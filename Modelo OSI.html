    Definição das Camadas

O que é o Modelo OSI?


O Modelo OSI, que do inglês Open System Interconnection, é um modelo de rede de computador que é divido em 7 camadas de funções, com o objetivo de ser um padrão, para protocolos de comunicação entre os mais diversos sistemas de uma rede local, garantindo a comunicação entre dois sistemas computacionais.



Como funciona cada camada?


Cada camada do Modelo OSI é responsável por uma determinada operação, sendo denominadas como:

7. Camada de aplicação;
6. Camada de apresentação;
5. Camada de sessão;
4. Camada de transporte;
3. Camada de rede;
2. Camada de enlace de dados;
1. Camada física.


Definição de camadas:

Camada Física - é camada que define as especializações elétricas e físicas dos dispositivos, portanto, ela quem define se a transmissão pode ser ou não realizada nos dois sentidos simultaneamente, seja por meio elétrico, ópticas, mecânico ou funcional para o meio físico.

Camada de Enlace de Dados (ou Ligação de Dados) - é camada que detecta e corrige erros que possam acontecer no nível físico, controlando o fluxo(recepção, delimitação e transmissão de quadros) e também estabelece um protocolo de comunicação entre sistemas diretamente conectados.

Camada de Rede - é camada que fornece os meios funcionais e de procedimento de transferência de comprimento varável de dados de sequências de uma fonte de acolhimento de uma rede para um host de destino numa rede, portanto ela realiza o roteamento de funções, e também pode realizar a fragmentação e remotagem e os erros de entrega de relatório.

Essa camada de rede pode ser dividida em três subcamadas:

++++Sub-rede de acesso - considera protocolos que lidam com a interface para redes;

++++Sub-rede dependente de convergência - necessária para elevar o nível de uma rede de trânsito, até ao nível de redes em cada lado;

++++Sub-rede independente de convergência - lida com a transferência através de múltiplas redes. Controla a operação da sub rede roteamento de pacotes, controle de congestionamento, tarifação e permite que redes heterogêneas sejam interconectadas.

Camada de Transporte - é camada responsável por receber os dados enviados pela camada de sessão e segmentá-los ara que sejam enviados a camada de rede, logo, ela cuida do controle de fluxo, ordenação dos pacotes e correção de erros, evitando, dessa forma, que as mensagens sejam entregues sem erros, sem perdas ou duplicações. Ela pode operar em dois modos:

++++Orientado à conexão - Mais confiável(entrega o TPDU) e garante a ordenação das informações, mas depende do overhead para realizar o serviço.

++++Não-Orientado à conexão - Menos confiável(não entrega o TPDU) e não garante a ordenação das informações, mas por se tratar em uma rede local, não depende do overhead para realizar o serviço.

Camada de Sessão - é a camada responsável pela troca de dados e a comunicação de hosts, dessa forma, em uma mesma página dois computadores podem se comunicar (semelhantemente aos jogos online).

Camada de Apresentação - é camada que converte o formato do dado recebido pela camada de Aplicação em um formato comum a ser usado na transmissão de dados que seja compreensivo para o protocolo.

Camada de Aplicação - é camada que corresponde às aplicações no topo da camada OSI que promoverão a interação da máquina do usuário com o destinatário.
